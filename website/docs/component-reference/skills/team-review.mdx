# Use `team-review` skill

<span className="badge badge--handbook-code-review">handbook-code-review</span>

Code review using three persistent named team members (reuse-reviewer, quality-reviewer, efficiency-reviewer) that stay alive for follow-up questions.

Creates a team with three named reviewers running in parallel. Unlike `/subagent-review`, team members persist after review — you can send follow-up messages to specific reviewers.

## When to Use This Skill

Use the `team-review` skill when you want to:

- In-depth code review with ability to ask follow-up questions
- Iterative review process across multiple rounds
- Track review progress via tasks per reviewer
